## 09 — 휴머노이드 3D (핵심 발전사)
사용자 피드백을 반복 반영하며 크게 진화:

1. **휴머노이드화**: 박스 생명체 → 9노드 골격(머리·몸통·양다리·양팔).
   물리를 Verlet + PBD(Gauss–Seidel)로 교체해 2족에서도 안정.
2. **흐물거림 잡기**: 마찰을 pos_prev에만 적용(뼈 제약 유지), 솔버 수렴 개선,
   근육 명령 저역통과 필터.
3. **고꾸라짐·쪼그림 해결**: 보상에 "몸통 수직도 × 다리 펴고 선 높이" 곱셈 도입
   → 꼿꼿이 서야만 점수. 무릎 각도 제한(과신전 방지) 추가.
4. **3D 전환**: 원근 카메라·그림자·그리드 렌더(07 투영 재활용), 창 최대화(SCALED|RESIZABLE).
5. **외형**: 졸라맨 선 → 원통형 금속 사지(폭 따라 음영 그라데이션) + 금속 관절 +
   로봇 머리(바이저/안테나) + 가슴 코어라이트. 색감 화이트메탈+시안.
6. **발 추가**: 11노드(발끝 추가)로 평발(뒤꿈치+발끝 2점 접지) → 점 위 휘청거림 제거.
7. **행동 4종**: 걷기 / 짐 옮기기(무거운 손+크레이트) / 복싱(샌드백·글러브·펀치) / 춤.
8. **실제 로봇 방식 도입**: 실제 휴머노이드(G1·Atlas·Optimus) 보행 특성 조사
   (직립·이중지지·낮은 발 클리어런스·팔다리 협응·ZMP/MPC·모션 프라이어).
9. **DeepMimic 모방학습**: 자연 보행 레퍼런스(2링크 IK)를 신경망이 모방하도록 보상.
   → 신경망(뇌)이 실제로 제어 = "살아있는 신경망" viz 활성, 학습 곡선 의미있게 상승.
10. **ES(진화전략)**: GA → OpenAI식 ES로 교체. 학습 곡선이 바닥→급상승하는 S커브로 극적.
11. **달리기 모드**: 같은 모방학습 + 달리기 레퍼런스(앞기울기·큰보폭·빠른케이던스·비행구간·큰 팔펌핑).
12. **자세 다듬기**: 좌우 스탠스 확대(엉덩이 너비), 목 길이 단축, 무릎 굽힘 방향/세기 조정.

## 14 — "걷기"가 아니라 "질질 끌기" → 보행 보상 개선 (※ 아래 셰이핑은 §15에서 대체됨)
> 주의: 이 절의 air-time/slip 보상 셰이핑은 **현재 코드(12_g1_ppo.py)에 없다.** §15에서
> DeepMimic 모방학습으로 대체돼 질질끌기 문제를 더 근본적으로 풀었다. 아래는 그 경위 기록.

12시간 런의 4~5h 클립에서 로봇이 **발을 안 떼고 미끄러지듯 전진**(학습곡선도 정체/하락). 코드 버그가 아니라
**보상이 '앞으로+안 넘어지기'만 줘서 생긴 전형적 local optimum**(발 떼면 초반 낙상으로 생존보상 손실 → 안전한 끌기 선택).
- 개선: **발 교대 딛기(feet air-time) 보상** + **발 미끄러짐(slip) 패널티** + **자세 페널티 완화(0.3→0.1)**.
  "끌기"를 직접 감점하고 발 들기를 보상해 진짜 스텝을 유도.
- 교훈(이 단계): 09에서 배운 "자연스러움은 보상으로 만든다"가 PPO에서도 동일 — 단순 전진보상은 반드시
  괴상한 보행으로 수렴하며, **발 접지/스윙을 직접 성형(reward shaping)** 해야 한다. RL 튜닝은 1회로 끝나지 않음.

## 15 — 12번(G1) 보상 방식 정리: air-time/slip 셰이핑 → DeepMimic 모방으로
§14는 "전진+생존" 보상의 질질끌기를 **air-time/slip 보상**으로 교정하려 했다. 그러나 현재
`12_g1_ppo.py`(G1Env)는 그 셰이핑 대신, **09에서 검증된 DeepMimic 모방학습을 실제 G1에 이식**한
형태로 정착했다. (레퍼런스 모션을 따라가게 하면 발 떼기가 처음부터 나와 "질질끌기 local optimum"을
보상 미세조정이 아니라 **구조적으로** 회피한다 — §14의 교훈을 한 단계 더 근본적으로 푼 셈.)
- **레퍼런스 `ref_pose`**: 고관절피치·무릎·발목피치를 사인으로 합성한 파라메트릭 보행 사이클
  (좌우 π 위상차, 팔은 같은쪽 다리와 반대 스윙, 좌우 체중이동 lean). 정책은 **잔차(ACTION_SCALE 0.6)** 만 학습.
- **관찰에 위상(cos,sin) 추가** → DeepMimic 표준(정책이 사이클 내 위치를 안다).
- **RSI**: 양발지지 위상(0.0/0.5 근처)에서만 시작 + **레퍼런스 관절속도로 qvel 시드**
  (속도 0으로 두면 사이클 중간 자세와 모순돼 시작 직후 낙상 — DeepMimic 표준).
- **보상**: `r_pose`(모방, POSE_W로 다리관절 가중) + `r_vel`(전진속도 V_REF 0.8 추종)
  + heading(정면) + up(직립) + alive(생존) + `r_rhythm`(스탠스=접지/스윙=공중) − |vy| − act_rate(잔차 떨림).
- **PPO 설정**: log_std_init=−1(잔차 탐험 노이즈 std 1.0→0.37; 29관절 풀스윙 노이즈로 균형신호가
  묻히는 것 방지), N_ENVS 6, net_arch[256,256], VecNormalize(obs+reward).
- **검증(실측)**: G1Env 인스턴스화 → NU 29 / OBS_DIM 71 == `_obs()` 실측 / `ref_pose` 29 ==
  qpos[7:] / 발링크 id 탐색 / RSI 후 골반 z≈0.75 / 랜덤 200스텝 전 구간 finite, 보상 [1.0, 4.0]
  (이론 최대 ≈4.4) 확인. 차원·인덱싱·수치안정 모두 정상.
- 주의(정직): air-time/slip을 끝까지 학습 비교한 기록은 없다. DeepMimic 이식이 더 자연스러운
  발 떼기를 **출발선부터** 보장한다는 설계상 이점이 채택 이유다.

- **검증**: 풀 G1·12-dof 모두 plate False/True에서 차원·총질량·관성 동일(+3 geom, contype=conaffinity=0).
  walk 헤드리스 스모크(50프레임 mp4) 통과. deploy는 동일 로직을 raw g1_12dof.xml로 검증
  (외부 legged_gym 실행 환경 자체는 미검증).

## 배운 점 (핵심 교훈)
- **단순 물리+GA로는 자연스러운 보행이 안 나온다** — GA는 점수만 높이는 괴상한 동작을 찾음.
- **자연스러움은 보상/레퍼런스로 만든다** — DeepMimic식 모션 프라이어가 결정적.
- **물리 디테일이 중요** — 마찰 적용 위치, 솔버 수렴(Jacobi→Gauss–Seidel), 발 접지면 등.
- 정직한 한계 인정: 토이 시뮬은 실제 로봇 제어기(MPC/실액추에이터)를 대체할 수 없음.
