# SPG S1 (Unitree G1) 보행 RL — 오프라인 실행 매뉴얼

이 폴더의 휴머노이드 보행 강화학습 프로그램을 **로컬 PC에서 Python으로** 실행하는 방법.

---

## 1. 준비 (한 번만)

- **Python 3.10 ~ 3.14** (3.14에서 동작 확인됨)
- 의존성 설치:

  ```bash
  pip install -r requirements.txt
  ```

- **첫 실행 시 인터넷 1회 필요**: `robot_descriptions`가 Unitree G1 로봇 모델(MJCF+메시)을
  자동으로 받아 캐시에 저장합니다. 이후로는 오프라인으로 돌아갑니다.

> GPU 없어도 됩니다(CPU로 동작). GPU가 있으면 렌더가 더 부드럽습니다.

---

## 2. 가장 쉬운 실행 — `python run.py`

설치 후 **한 줄**로 바로 본다. 학습된 정책이 없어도 **자연 보행 레퍼런스(AI가 모방할
목표 걸음새)를 즉시 재생**하므로, pip install 직후 또박또박 걷는 모습을 볼 수 있다.

```bash
python run.py                 # 자연 보행 레퍼런스 재생(즉시·체크포인트 불필요)
python run.py train           # 빈 정책부터 학습하며 관찰
python run.py play            # ./checkpoints 의 학습된 정책 재생
python run.py play 받은폴더    # 다른 폴더의 정책 재생
python run.py reference --record walk.mp4 --seconds 15   # 창 없이 mp4 저장
```

아래는 각 모드를 직접 부르는 방법(run.py가 내부에서 이걸 호출한다).

## 3. 실행 모드 상세

### A. 학습하면서 보기 — `python 12_g1_ppo.py`
빈 정책에서 시작해 **실시간으로 PPO 학습**하며, 로봇·살아있는 신경망·학습곡선을 창으로 보여줍니다.

```bash
python 12_g1_ppo.py
```
- 조작: `↑↓` 학습집중(렌더 fps 조절) · `Q/E` 카메라 회전 · `R` 리셋 · `ESC` 종료
- 처음엔 못 걷는 게 정상입니다(학습 초기). 오래 둘수록 좋아집니다.

### B. 학습된 정책 재생 — `python play.py <체크포인트폴더>`
이미 학습된 정책(체크포인트)을 불러와 **결과만 재생**합니다. 학습 안 하므로 가볍고,
클라우드(GitHub Actions)에서 학습한 정책을 받아 로컬에서 확인하기 좋습니다.

```bash
python play.py checkpoints          # ./checkpoints 의 정책 재생
python play.py 받은체크포인트폴더    # 다른 폴더 지정
```
- 체크포인트 폴더에는 **`ppo_model.zip` 과 `vecnormalize.pkl`** 이 함께 있어야 합니다.
- 조작: `R` 리셋 · `Q/E` 회전 · `↑↓` fps · `ESC` 종료

### C. 창 없이 mp4로 녹화 (공유·검증용) — `python play.py <ckpt> --record`
디스플레이 없이(서버/헤드리스) 정책 재생을 mp4로 저장합니다.

```bash
python play.py checkpoints --record walk.mp4 --seconds 15
```

---

## 3. 학습된 체크포인트 구하기

`play.py`로 보려면 학습 산출물(`ppo_model.zip` + `vecnormalize.pkl`)이 필요합니다.

- **클라우드 학습 결과 받기**: GitHub 저장소 → **Actions** 탭 → 해당 학습 run →
  아래 **Artifacts**의 `checkpoint-final`(또는 `checkpoint-after-segment-1`) 다운로드 →
  압축 풀어서 그 폴더를 `play.py`에 지정.
  - 또는 학습 중 미리보기 클립은 저장소 **Releases**의 `live-run<번호>`에서 바로 볼 수 있습니다.
- **직접 학습**: 위 A(`12_g1_ppo.py`)를 오래 돌리거나, 헤드리스로
  `python tools/record_training.py` (환경변수 `CKPT_DIR`로 저장 폴더 지정).

---

## 4. 헤드리스 학습 + 자동 녹화 (선택, 고급)

화면 없이 연속 학습하며 일정 간격으로 클립을 저장(서버/CI용):

```bash
# Linux 등 헤드리스: 소프트웨어 GL 필요 (sudo apt-get install libosmesa6)
CKPT_DIR=checkpoints OUT_DIR=recordings DURATION_SEC=3600 INTERVAL_SEC=600 \
  python tools/record_training.py
```
주요 환경변수: `DURATION_SEC`(이번 세그먼트 학습 시간), `INTERVAL_SEC`(녹화 간격),
`CLIP_SEC`(클립 길이), `CKPT_DIR`/`OUT_DIR`(저장 위치). 체크포인트를 저장/복원해 이어학습됩니다.

---

## 5. 문제 해결

| 증상 | 해결 |
|---|---|
| `ModuleNotFoundError` | `pip install -r requirements.txt` 다시 실행 |
| 첫 실행이 멈춘 듯 함 | G1 모델 다운로드 중입니다(인터넷 1회 필요). 잠시 대기 |
| 창이 안 뜸 / GL 오류(서버) | 헤드리스면 B 대신 C(`--record`)를 쓰거나, Linux는 `libosmesa6` 설치 후 `MUJOCO_GL=osmesa` |
| 한글이 깨짐 | 시스템에 한글 폰트(맑은 고딕 등)가 있으면 자동 사용. 없으면 라벨만 영문 대체 |
| `체크포인트를 찾을 수 없습니다` | 지정 폴더에 `ppo_model.zip` + `vecnormalize.pkl` 둘 다 있는지 확인 |

## 파일 안내

- `12_g1_ppo.py` — 메인 프로그램(학습 + 실시간 관찰)
- `play.py` — 학습된 정책 재생 뷰어(오프라인)
- `tools/record_training.py` — 헤드리스 학습 + 자동 녹화
- `tools/eval_policy.py` — 정책 보행 메트릭 평가(전진속도/생존/낙상율)
- `requirements.txt` — 의존성
